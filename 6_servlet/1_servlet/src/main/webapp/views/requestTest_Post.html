POST방식으로 요청해보기

특징1. POST방식으로 요청시 URL의 BODY영역에 데이터를 포함시켜 넘긴다.
=> 사용자가 입력한 값(데이터)들이 url에 노출되지 않는다.
=> 보안유지가 가능하다
=> 즉, 로그인이나 회원가입 같은 경우 POST방식에 적합함 특징2.Body영역은 전송하는 길이에 제한이 없다. => 즉, 게시판작성같은 경우 POST방식이 적합하다.

특징3. 즐겨찾기는 가능하나 데이터가 url에 노출되어있지 않다. =>즉, 기존에 봤던 응답화면을 볼 수 없을 수 있다.
=>즉, 검색 기능 같은 경우 POST방식에 부적합 하다.
특징4. Time out이 존재하기 때문에 페이지요청, 기다리는 시간이 있다.
